# Changelog

## [2026-03-10]

### Added
- **ACL の複数 API キー対応**  
  `x-api-key` ヘッダーまたはクエリパラメータにカンマ区切りで複数の API キーを指定できるようになりました。指定されたすべてのキーの許可ルートフォルダがマージ（和集合）されます。
- **カスタムテキスト拡張子のサポート**  
  `extra_text_extensions` パラメータを追加し、`.conf`, `.ini`, `.env`, `.log` などの任意のテキスト形式ファイルを検索対象に含められるようになりました。

### Fixed
- **Ollama v0.17.6+ での `glm-ocr` 互換性**  
  Ollama v0.17.6 以降で `glm-ocr` の不具合が修正されたことを確認しました。これに伴い、警告の修正と動作確認済みバージョンの情報を更新しました。
- **SSE ヘッダーの日本語文字化け（Mojibake）対応**  
  CLINE などのクライアントから送られる非 ASCII（日本語）文字を含むヘッダーが正しくデコードされない問題を修正しました。

### Improved
- **インデックス同期の進捗表示の改善**  
  `get_sync_status` ツールで、OCR 変換のページ数やパーセンテージ、DB 登録中のファイル名など、より詳細な進捗状況を確認できるようになりました。

### Changed
- **依存パッケージの汎用化と Python 3.13 対応**  
  `requirements.txt` のバージョン指定を緩和し、Python 3.13 や Numpy 2.0 以降の環境でも動作可能なように調整しました。
- **ドキュメントの更新**  
  `extra_text_extensions` の説明や最新の不具合修正情報を README に追記しました。

---


## [2026-03-04]

### Added
- **PaddleOCR 対応を追加**  
  代替 OCR エンジンとして **PaddleOCR** のサポートを追加しました。  
  `config.json` の `ocr_engine` パラメータで `"ollama"` と `"paddleocr"` を切り替えられます。  
  PaddleOCR の CPU / GPU 切り替えは `paddleocr_use_gpu` パラメータで制御できます。
- **OCR の異常出力の自動検知機能**  
  OCR の結果が空白、極端に短い文字列、または同一文字の異常な繰り返しになる問題を検知し、OCR処理を失敗（空文字として扱う）として安全にフォールバックする機能を追加しました。
- **PaddleOCR GPU セットアップガイドの追加**  
  GPU 環境で PaddleOCR を動作させるための手順書として `PADDLE_GPU_SETUP.md` を新規追加しました。

### Changed
- **README.md アーキテクチャ図の改善**  
  アーキテクチャ図に「ユーザー側」と「サーバー側」の境目を明記し、どこからどこまでがシステムの管理範囲なのかを視覚的に分かりやすく改善しました。
- **Ollama v0.17.x 系統への注意喚起の追加**  
  v0.17.x の `glm-ocr` 互換性問題（OCRエラー）に関する注意喚起と、ダウングレードや PaddleOCR への切り替え等の回避策を README に記載しました。
- **依存パッケージの更新**  
  `requirements.txt` に `paddleocr==2.7.3` および競合回避のための `numpy<2.0.0` を追加しました。
